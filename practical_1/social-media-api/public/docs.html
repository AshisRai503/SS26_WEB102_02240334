📱 Social Media API Documentation

Welcome to the Social Media API - a RESTful API for social media platform functionality.

Base URL: http://localhost:3000/api
Note: For protected endpoints (POST, PUT, DELETE on posts), include header: X-User-Id: 1

👤 Users

GET /users

Get a list of all users with pagination

Query Parameters

Parameter Type Description
page Number Page number (default: 1)
limit Number Items per page (default: 10)

Example Response

{
  "success": true,
  "count": 3,
  "page": 1,
  "total_pages": 1,
  "data": [
    {
      "id": "1",
      "username": "traveler",
      "full_name": "Karma",
      "profile_picture": "https://example.com/profiles/traveler.jpg",
      "bio": "Travel photographer exploring the world",
      "created_at": "2023-01-15"
    }
  ]
}
GET /users/:id

Get a specific user by ID

Example Response

{
  "success": true,
  "data": {
    "id": "1",
    "username": "traveler",
    "full_name": "Karma",
    "profile_picture": "https://example.com/profiles/traveler.jpg",
    "bio": "Travel photographer exploring the world",
    "created_at": "2023-01-15"
  }
}
POST /users

Create a new user

Request Body

{
  "username": "new_traveler",
  "email": "new@example.com",
  "full_name": "New Traveler",
  "bio": "Adventure seeker"
}

Example Response

{
  "success": true,
  "data": {
    "id": "4",
    "username": "new_traveler",
    "full_name": "New Traveler",
    "email": "new@example.com",
    "profile_picture": "default-profile.jpg",
    "bio": "Adventure seeker",
    "created_at": "2023-03-20"
  }
}
PUT /users/:id

Update an existing user

Request Body

{
  "full_name": "Updated Name",
  "bio": "Updated bio"
}
DELETE /users/:id

Delete a user

📝 Posts

GET /posts

Get a list of all posts with pagination

Query Parameters

Parameter Type Description
page Number Page number (default: 1)
limit Number Items per page (default: 10)
GET /posts/:id

Get a specific post by ID

POST /posts

Create a new post (requires authentication)

Headers

X-User-Id: 1

Request Body

{
  "caption": "My new post!",
  "image": "https://example.com/images/new.jpg"
}
PUT /posts/:id

Update a post (requires authentication - must be post owner)

Headers

X-User-Id: 1
DELETE /posts/:id

Delete a post (requires authentication - must be post owner)

Headers

X-User-Id: 1

📡 Testing the API

You can test the API using:

  • Postman - Download from postman.com
  • curl - Command line tool
  • Browser - For GET requests only

Example curl commands:

# Get all users
curl http://localhost:3000/api/users

# Get user with ID 1
curl http://localhost:3000/api/users/1

# Create a new user
curl -X POST http://localhost:3000/api/users \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@example.com","full_name":"Test User"}'

# Get all posts
curl http://localhost:3000/api/posts

# Create a post (requires X-User-Id header)
curl -X POST http://localhost:3000/api/posts \
  -H "Content-Type: application/json" \
  -H "X-User-Id: 1" \
  -d '{"caption":"My new post","image":"https://example.com/image.jpg"}'

🔧 Error Responses

📊 Status Codes

  • 200 OK - Request successful
  • 201 Created - Resource created successfully
  • 400 Bad Request - Invalid request
  • 401 Unauthorized - Authentication required
  • 404 Not Found - Resource not found
  • 500 Server Error - Internal server error
